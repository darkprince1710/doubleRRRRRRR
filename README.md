# Алгоритм Крускала для минимального остовного дерева

## Задача
Реализация алгоритма Крускала с использованием жадной стратегии для нахождения минимального остовного дерева в взвешенном неориентированном графе.

## Требования
- Использовать структуру Union-Find для проверки циклов
- Входные данные: граф с весами ребер
- Выходные данные: список ребер минимального остовного дерева и общий вес

## Описание алгоритма
Алгоритм Крускала работает по следующему принципу:
1. Сортируем все ребра графа по весу в порядке возрастания
2. Последовательно добавляем ребра в остовное дерево, начиная с наименьшего веса
3. Используем Union-Find для проверки, не создает ли добавляемое ребро цикл
4. Продолжаем до тех пор, пока не будет выбрано V-1 ребро (где V - количество вершин)

## Структура данных
- **Union-Find**: для эффективного определения принадлежности вершин к компонентам связности
- **Структура Edge**: хранит информацию о ребре (источник, назначение, вес)
- **Структура Graph**: содержит количество вершин, ребер и список всех ребер

## Входные данные
Граф с 6 вершинами и 9 ребрами с весами:
- Вершины: 0, 1, 2, 3, 4, 5
- Ребра с весами представлены в виде (источник, назначение, вес)

## Результат работы
Программа выводит:
- Список ребер минимального остовного дерева
- Общий вес построенного остовного дерева

## Сложность
- Временная сложность: O(E log E) из-за сортировки ребер
- Пространственная сложность: O(V + E) для хранения графа и структур данных
